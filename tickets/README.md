# Feature Tickets

This directory contains feature planning tickets for the Zuora SE Demo Data Agent.

## Workflow

1. **Plan** — A new ticket is created in `draft` status with a problem statement, proposed solution, and implementation plan.
2. **Review** — Discuss and refine the ticket until the approach is solid.
3. **Approved** — Mark the ticket `approved` to signal it's ready to build.
4. **In Progress** — Work begins; the ticket moves to `in-progress`.
5. **Done** — Implementation is complete and deployed.

## Ticket Format

Each ticket is a markdown file named `XXXX-short-description.md` with a structured header and sections.
